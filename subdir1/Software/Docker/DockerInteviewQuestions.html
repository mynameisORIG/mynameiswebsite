Docker Interview Questions

What's a hypervisor

It's a computer softare/firmware/hardware that creates and runs a Virtual Machine

What's virtualization

It's the process creating software-based, virtual version of soemthing (computer storage, servers, applications, etc.) These virtual environments create a single physical hardware system.

What's conatinerization

It's an app that's being develpoed and deployed that/s bundles and wrapped together with configuration files and dependencies.

What's a Docker Container

A docker container inclues the application and all it's dependencies. It also shares the kernel with other containers, running isolated processes in user space on the host Operating System.

What's a DockerFile

Docker can build images automatically by reading instructions from file called Dockerfile. A dockerfile is a text document that contains commands a user could call on the comand line to assemble an image

What's Docker Compose

It's a YAML file which contains details about services, networks, and volumes for setting up a Docker application.

What's a Docker Swarm

It's a native clustering for Docker. It turns a pool of Docker hosts into a single, virtual Docker host.

What's a docker Namespace

A namespace is one of Linux's features and a important conecept of containers. Namespaces add layer of isolatino in containers.

What's a docker machine

A docker machine is a tool that lets you install Docker Engine on virtual hosts.

How can docker be used

Docker can be used by:

  1. simplifying configuration
  2. Code pipeline management
  3. Development productivity
  4. Application isolation
  5. Debugging Capabilites
  6. Multi-tenacy
  7. Rapid Deployement

How is Docker different from other containerized methods

Docker containers are easy to deploy any cloud platform. It gets more applications runningn on the same hardware when compated to other technology.

What are the different states of docker containers

Created, Running, Restarting, Exited, Dead

Can you remove a pause container in docker

No, the container needs to be stopped before it is removed.

Will Docker compose wait for a current container to be ready to move to a running state next service

Docker compose always runs in dependency order
